Template-Driven Forms

Excercise 1

In this example we want to create a simple template-driven form. For registering for a new website. There is a few steps we need to undertake.

  1. Take a look at the app.component.ts you will find an interface User, which contains all the data fields a newly registered user might require. Its properties are the variables we want to set using the form.
  2. As we want to use forms in this exercise, the FormsModule needs to be imported and injected into our app. Head over to the app.module.ts and uncomment the necessary lines there.
  3. The current inputs do not have types assigned to them. Assign the correct type="xxx" to each input. Head over to mozilla.org to see which types are available.
  4. In this form all input values except the name are mandatory. Mark all mandatory inputs as such using the required attribute. If you need to refresh your memory head over to mozilla.org
  5. Each input of a template-driven form needs to be named. Assign the name="xxx" property for each input.
  6. Wrap all questions inside a form element. The form element should recieve the attribute #signupForm="ngForm", which sets its id to #signupForm and declarates it as an ngForm.
  7. Remember the user we had a look at in in 1.? Use two-way-data-binding with [(ngModel)] to bind the value of each of the inputs to their respective property.
  8. If you did everything right up to this point, the properties of user should update, when you type something into an input and you should be able to use {{signupForm.valid}} to show whether the form has been correctly filled in. You can uncomment the respective line in the code.
  9. To finish up, it would be great to be able to submit the form. The button is already available. Make sure to mark it as the submission button by adding the property type=submit to it. We also want to disable the button, until the form is valid adding [disabled]="!signupForm.valid" should do the trick. Now we only need to let our form know what to do, when the submit button is clicked by adding (ngSubmit)="onSubmit()" to its properties.
